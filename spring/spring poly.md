
Spring -> 객체지향을 도와주는 프레임워크 

### 다형성 

유연, 변경 용이 
변경과 구현을 구분 

클라이언트는 대상의 인터페이스만 알면됨 

CLIENT ----------> INTERFACE  (역할)
					| 
					|
				IMPLEMENTS (구현)
			객체 A, 객체 B, 객체 C .... 

CLIENT 입장에서는 역할만 알고있으면 구현된 객체를 쓸수있다. 
객체가 추가되던 변경되던 INTERFACE를 상속받았으니 CLIENT는 구현된 모든 객체를 사용할수있다. 


save() 요청 --------> interface save() 
					  |
					  |
				@Override

클라이언트를 변경하지않고 서버 구현 기능을 유연하게 변경 가능

ex) 
클라이언트가 기존 하드에 저장하던 데이터를 서버에 저장하고 싶어짐 
-> 그냥 save() 를 서버에 저장하는 클래스를 구현해 Override 시키면 클라이언트는 
의존성주입을 통해 클라이언트 코드 변경없이 기능을 확장 가능 

인터페이스 설계 중요 

스프링 --> IoC , DI 지원 --> 구현을 편리하게 변경할수있음
