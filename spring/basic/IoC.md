
#### IoC Inversion of control 

구현객체가 프로그램 실행 제어를 하는게 아닌 다른 객체를 사용해 흐름을 맡기는걸 Inversion of control 이라고한다. 

#### 프레임워크 vs 라이브러리 

프레임워크는 내가 작성한 코드를 제어한다. 
라이브러리는 내가 작성한 코드가 제어의 흐름을 담당한다 

#### DI

의존관계는 정적 , 동적인 의존관계로 나뉜다. 

정적의존관계는 import 로 명시적으로 적혀있는 의존관계로 애플리케이션을 실행하지 않아도 분석할수있음 
이러한 의존관계는 실제 런타임때 인터페이스에 무슨 객체가 할당될지 모름 

동적인 의존관계는 애플리케이션이 실행된 시점에 실제 생성된 인스턴스의 참조가 연결된 의존관계 

실제 런타임에 외부에서 실제 구현 객체를 생성해 클라이언트에 전달해 클라이언트와 서버의 실제 의존관계가 연결되는걸 DI 라고 함 

DI를 사용하면 동적의존관계를 쉽게 바꿀수있다. 

객체를 생성하고 관리해주는 걸 

IoC 컨테이너 또는 DI 컨테이너라고 한다.
