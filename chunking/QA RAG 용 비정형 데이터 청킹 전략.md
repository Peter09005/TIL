해당 HTML 은 기존 상담원이 대출 상품을 추천하기 위해 쓰던 표 

상담용 의사결정 테이블 
즉 상담원이 고객정보를 입력하면 -> 바로 대출가능/ 조건/ 한도 등을 판단하는 역할이였음

```
입력값 -> 판단조건 -> 결과 

상담로직으로 해석해야되는거 같음
```

~~~
즉 고객의 상태 = { 
	지역,
	자금용도,
	주택보유수,
	상애최초 여부 
	....
}

상태 -> 결과 

결과 -> 상담 멘트 / 승인 여부 를 판단해야함.
~~~

일반적인 RAG 라면 

~~~ 
질문 -> 관련 문서 검색 -> 요약 답변 
~~~

하지만 지금 목표는 
~~~
질문 → 상담 유형 분류 → 필요한 조건값 추출 → 판정표 row 검색 → 예외/FAQ 확인 → 상담 답변
~~~

~~~
LV1 = 상담 주제/탭/업무 상황
LV2 = 상담원이 눌러보는 답변 블록
표 = 상담 판정 매트릭스
FAQ/TIP = 상담 중 예외질문 대응용 보조 카드
[확인]/[공문] = 상담원이 필요시 클릭하는 내부 확인 버튼
~~~

으로 판단해봐야겠음 

~~~
LV1 안에 포함된 것만 기본으로 사용
LV1 밖으로는 자동 확장하지 않음
중복 청크는 canonical로만 관리
~~~

LV1 의 관계를 연결하는거 보다는 상담원이 보는 순서를 보존하는게 중요. 
LV1 은 지식구조가 아니라 UI 구조였음. 

# 질의 입력 

~~~
"서울에 집 사려고 하는데 대출 얼마나 나와?"
~~~


# 목표 

상담원이 이 표를 보고 하던 판단을 AI 가 재현하게 만들기 

1. 질문을 조건으로 바꾸기 
2. 조건에 맞는 chunk 검색 
3. 근거 기반 답변 
	


~~~
input: 서울에 집 사려는데 주담대 얼마나 나와?

	→ 상담 의도 분류  
	→ 필요한 슬롯 추출  
	→ 부족한 슬롯 확인 질문  
	→ 판정표/FAQ 조회  
	→ eligibility + limit 결과 생성

output: 서울은 규제지역 기준으로 봐야 합니다. 한도 판정을 위해 주택보유수, 주택가격, 생애최초 또는 서민실수요자 해당 여부가 필요합니다.
~~~

~~~
FAQ / TIP 

상담 중 나오는 애매한 케이스 해석 
~~~

~~~
effective date 

9월에 신청했는데 지금 실행해요
기존 승인 건인데 10.16 규제 적용돼요?
25.06.28 전에 접수한 건은요?
예전 기준으로 안내받았는데 바뀐 건가요?

기본값이 아니라, 기존 승인 건 / 경과 규정 이런걸 물어볼때
~~~
